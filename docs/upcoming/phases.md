# Implementation phases

The master prompt's order is authoritative:

1. Foundation
2. Chess domain
3. Core UI
4. Computer opponent
5. Local multiplayer
6. Friend multiplayer and relay
7. Challenges and economy
8. Practice, guide, saves, and review
9. Settings and developer options
10. Localization
11. Legal, open source, and documentation completion
12. QA and release preparation

Core chess rules and perft tests must pass before work is represented as moving
to later product features. Cross-cutting safety fixes may be made at any time.
